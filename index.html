¿Cómo funciona el cifrado con Lorenz?

Este sistema cifra mensajes usando las matemáticas del caos. Descubre el paso a paso.

🔑 Paso 1 — La Clave Inicial

Todo comienza con tres números decimales, por ejemplo: (5.0, 1.212, 2.3548). Más que una contraseña, son el "punto de partida" de nuestro sistema caótico.

Ver rangos permitidos
ParámetroRango Válido
x₀Entre −20 y +20
y₀Entre −30 y +30
z₀Entre 0 y 50
🦋 El Efecto Mariposa: Cambiar la clave en tan solo 0.0001 produce resultados de cifrado completamente distintos.

🔢 Paso 2 — Mensaje a Números

Los computadores no entienden letras. Usamos el estándar ASCII para convertir tu mensaje a números.

"Hola" ➔ [72, 111, 108, 97]

⚙️ Paso 3 — El Motor Caótico (RK4)

Las ecuaciones de Lorenz dictan cómo se mueve nuestro sistema. Como no tienen solución exacta, usamos un método matemático súper preciso llamado Runge-Kutta (RK4) para avanzar paso a paso sin perder el control del caos.

Ver las fórmulas matemáticas de Lorenz
dx/dt = 10 · (y − x) dy/dt = x · (28 − z) − y dz/dt = x · y − (8/3) · z
Ver cómo funciona el RK4 (Estimación de Pendientes)

El RK4 calcula cuatro pendientes y las promedia para máxima precisión:

  • k₁: Inclinación inicial.
  • k₂, k₃: Inclinaciones intermedias.
  • k₄: Inclinación final.
yₙ₊₁ = yₙ + (1/6) · (k₁ + 2k₂ + 2k₃ + k₄)
Ver un ejemplo numérico real del RK4
// Partiendo de x₀=1.0, y₀=1.0, z₀=1.0, h=0.01 // 1. Calculamos k1 k₁ₓ = 0.0, k₁ᵧ = 0.26, k₁_z = −0.01667 // 2. Calculamos k2 k₂ₓ = 0.013, k₂ᵧ = 0.258, k₂_z = −0.01522 // 3. Calculamos k3 k₃ₓ = 0.01225, k₃ᵧ = 0.25773, k₃_z = −0.01487 // 4. Calculamos k4 k₄ₓ = 0.02455, k₄ᵧ = 0.25499, k₄_z = −0.01299 // 5. Estado final tras la iteración (Promedio) x₁ ≈ 1.01222 y₁ ≈ 1.25808 z₁ ≈ 0.98497

🧬 Paso 4 — Extracción del Keybit

Tras cada iteración del RK4, transformamos las nuevas coordenadas en un número entre 0 y 255. ¡Este es nuestro Keybit secreto!

KeyByte = Math.floor( (|x| + |y| + |z|) * 1e8 ) % 256
Ver ejemplo numérico

1. Sumar valores absolutos: |1.012| + |1.258| + |0.984| = 3.255

2. Escalar: 3.255 × 10⁸ = 325527000

3. Módulo 256: 325527000 mod 256 = 152 (¡Este es el keybit!)

🛡️ Paso 5 — Cifrado XOR Final

Comparamos el número de nuestra letra (Paso 2) con el Keybit (Paso 4) usando la operación XOR (O Exclusivo) bit a bit.

💡 ¿Qué es XOR? Es una regla criptográfica matemática muy sencilla: si tienes dos bits iguales, el resultado es 0. Si tienes dos bits diferentes, el resultado es 1. Imagínalo como un interruptor de luz reversible: si el Keybit es 1, "invierte" el bit del mensaje. Si es 0, lo deja igual. Esta operación es perfecta porque si aplicas el interruptor dos veces con la misma clave, los bits vuelven mágicamente a su estado original.
Letra 'H' (72) ⊕ Keybit (182) = Cifrado (254)

Para descifrar, el receptor hace el mismo proceso inverso: 254 ⊕ 182 = 72 ('H').

🔄 Bucle: El sistema repite los pasos del 3 al 5 para cada letra. Como el sistema caótico siempre avanza, cada letra tiene su propio Keybit único.
